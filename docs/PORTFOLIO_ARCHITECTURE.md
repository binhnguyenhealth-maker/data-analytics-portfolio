# PORTFOLIO_ARCHITECTURE.md

**SYNTHETIC DATA / INDEPENDENT PORTFOLIO PROJECT**

Architecture of the three-case-study public portfolio repository.

## Design principles

1. **Synthetic by construction.** Every dataset is generated by deterministic
   Python code from a single root seed (`shared/synthetic/__init__.py`).
   No production, client, or third-party data is present.
2. **Pattern, not payload.** The repository demonstrates analytical patterns
   (governed pipelines, leak-free benchmarks, explainable peer selection,
   stability layers, non-map fallbacks) rather than specific findings.
3. **Fail-closed validation.** Each project ships positive and negative
   fixtures. Negative fixtures mutate a base case and assert a specific
   rejection code; weakening a test to obtain a pass is prohibited.
4. **Disclosure-first.** Every project README carries a visible
   `SYNTHETIC DATA / INDEPENDENT PORTFOLIO PROJECT` label and lists what
   the project does and does not prove.
5. **Single-root presentation.** Two of the three projects (urban-mobility-gap
   and peer-scenario-explorer) deliberately use a single pre-joined physical
   presentation table for the dashboard surface; the third (data-quality-
   command-center) uses a governed one-to-many relationship with a declared
   unique-key left endpoint.

## Seed protocol

```
ROOT_SEED (single documented constant in shared/synthetic/__init__.py)
   |
   +-- derive_seed(project_id, purpose)
         SHA-256(root_seed | project_id | purpose) -> 64-bit integer
         |
         +-- make_rng(project_id, purpose) -> independent random.Random
         +-- snapshot_id(project_id) -> stable per-project snapshot label
```

Two regenerations from the same source tree produce byte-identical CSV/JSON
outputs. The `regenerate_all.py` driver runs every generator; the shared
determinism test rebuilds each project in **fresh interpreter subprocesses**
under at least three distinct `PYTHONHASHSEED` values and compares complete
output-byte manifests (path → SHA-256). A same-process reload cannot detect
per-process hash-seed drift, so the runtime proof is subprocess-isolated; a
static check additionally forbids the built-in `hash()` anywhere in a
generator or shared helper.

## Project map

| Project | Pattern | Presentation model | Key invariant |
|---|---|---|---|
| data-quality-command-center | governed refresh + priority queue | one-to-many `issue -> lineage` on `issue_id` (unique left) | total lexicographic priority order; no blended score |
| urban-mobility-gap | expected-vs-actual diagnostic | single-root pre-joined presentation table | modal shares sum to 1.0; equality guide shares scatter axes |
| peer-scenario-explorer | explainable peer selection + stability | single-root presentation with 4 surface values | exactly 5 peers; distance decomposition reconciles; non-map fallback |

## Layered responsibilities

```
shared/synthetic/        fictional entity pools + deterministic primitives
shared/validation/       regenerate, test, manifest, disclosure + structure scans
projects/<p>/src/        per-project generator (the only place data are made)
projects/<p>/tests/      per-project positive + negative fixtures
projects/<p>/data/synthetic/   generated outputs (gitignored; reproducible)
docs/                    architecture, disclosure matrix, Tableau rebuild plan
validation/              inclusion manifest (SHA-256) and validation record
projects/<p>/tableau/     packaged Tableau workbook
projects/<p>/images/      native Tableau preview
```

## Release boundary

- Exactly one packaged Tableau workbook and one native preview are included per
  project. All other native analytics and office-document formats remain
  denied by the structure gate.
- Apache License 2.0 applies to the repository.
- No production, client, or third-party data, names, codes, or identifiers.
- No derivation from, or reference to, any client deliverable, solicitation
  response, procurement engagement, or production deployment. This tree is a
  standalone synthetic prototype; see `DISCLOSURE_MATRIX.md`
  "Boundary with other work".

## Extension points

- A later Tableau revision must continue to consume the synthetic CSV/JSON
  outputs and satisfy the field contracts in `FUTURE_TABLEAU_REBUILD_PLAN.md`.
- A future scikit-learn reference model could replace the disclosed-coefficient
  benchmark in the mobility-gap project; the leak-free invariants must be
  preserved.
- A future hosted deployment would add a real refresh contract; the synthetic
  generator remains the local-only development reference.
