# Source and Usage Policy

**SYNTHETIC DATA / INDEPENDENT PORTFOLIO PROJECT**

## Source boundary

Every dataset under `projects/*/data/synthetic/` and `shared/synthetic/` is
deterministically generated by the Python code in this repository. Cities,
countries, source inventories, indicators, dates, and identifiers are
fictional and do not describe a real person, company, government, transit
agency, or engagement.

The packaged Tableau workbooks contain only the corresponding generated CSV
files. They do not contain production data, client records, third-party source
rows, access material, private contacts, or unnecessary personal information.

## Analytical-pattern boundary

The case studies are generic, independent re-implementations of common
analytics techniques: data-quality triage, benchmark-gap diagnostics, and
explainable peer-scenario analysis. They are portfolio prototypes and are not
evidence of paid delivery, client acceptance, production operation, hosted
Tableau administration, training, or measured business outcomes.

## Reproducibility

`shared/validation/regenerate_all.py` regenerates the synthetic data locally.
The release contract requires byte-identical outputs across fresh Python
processes and multiple hash-randomization seeds. Packaged workbooks are checked
against those exact generated bytes.

## Permitted public artifacts

The public repository may contain:

- source code, tests, documentation, and validation records;
- one packaged Tableau workbook under each `projects/*/tableau/` directory;
- one native Tableau preview under each `projects/*/images/` directory; and
- deterministic inclusion evidence under `validation/`.

Any additional native analytics file, archive, generated binary, production
source, real organization reference, or private identifier fails the release
gate unless the policy is explicitly reviewed and updated first.
